API Platform automated publishing

The API Platform running in the MDTP Platform has about 100 APIs. These expose over 500 different endpoints. They are consumed by third party software developers as well as other government departments (OGDs).

We are continually onboarding new APIs onto the platform, as well as updating existing APIs with new versions and functionality. Typically there are many releases of different APIs per week. It would be very hard to manually publish these changes to the API catalogue and make sure it is up-to-date.

So automation was added to the API platform. When a new version of an API is released onto the API platform it automatically updates the entry in the API catalogue - so that is always up to date!

So what does this mean?

  • when an API is deployed it is published into the API catalogue
  • the API teams don’t have to do anything
  • encourages change and innovation
  • the content in the API catalogue can’t be out of date

How does it technically work?

The API Platform has a CI/CD pipeline as a Jenkins job. As part of this process, it converts the APIs RAML to an OpenAPI specification (Swagger) definition file and publishes it into the API catalogue via the publish API (documented here).
